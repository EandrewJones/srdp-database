Hello, administrator!

Welcome to {{ config["COVER_NAME"]}}'s admin portal. Here you can perform a number of basic functions for site management and content moderation.

The Users and Posts dropdowns in the navbar will take you to an interface for performing basic CRUD operations on the database models. Every table can be sorted and filtered by column. Searchable columns vary by model.

Static Files and S3 Bucket offer some basic file management.

Users

User

The model representing users. Each user has a unique id and username. If known, either can be used to isolate entries. Users are searchable by id, username, name, email, and about me.

View model »

Likes

The relational model representing connections between users and posts. Entries are uniquely identified by the User-Post pair and should be isolated by User.Id and Post.Id. Likes are searchable by user's id, user's username, user's name, post id and post body.

View model »

Follow

The relational model representing connections between two users. Entries are uniqely identified by the Follower-Followed pair and should be isolated by Follower.Id and Followed.Id. Follows are searchable by id, username, name for follower and followed.

View model »

Posts

Post

The model representing posts. Each post has a unique id. If known, this can be used to quickly isolate a post. Posts are searchable by id, author's username, and body.

View model »

Comments

The relational model for representing a parent-comment connection between two posts. Comments themselves are a special subclass of posts. Each entry is uniquely identify by a Parent.Id-Comment.Id pair. This pair must be used to isolate entires. Comments are searchable by id and body for both parent and comment.

View model »

Reposts

The relational model for representing a root-repost connection between two posts. Reposts are a special subclass of posts that contain a reference to a 'root' post which is the embedded within the body of the repost. Each entry is uniquely identified by a Root.Id-Repost.Id pair. This pair must be used to isolate entries. Reposts are searchable by id and body for both root and repost.

View model »

File Management

Static Files

A directory-like GUI for simple static file management. If you want to change either background or logo post-launch, you may re-upload either with names matching those specified in the `.env` file. If the names are not the same, this will not work.

Warning: do not delete / modify either the `/src` or `/meplayer` folders!

View Static Files »

S3 Bucket

A lightweight GUI wrapper displaying the contents of the AWS S3 bucket for multimedia content. The website only has read/write privileges to the bucket, so cannot delete content. If you want to delete malicious multimedia content, you should delete the corresponding post or user account.
